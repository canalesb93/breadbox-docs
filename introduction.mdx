Breadbox is an open-source, self-hosted, financial aggregator for households. It manages connections to your banks via a provider like [Plaid](https://plaid.com/) or [Teller](https://teller.io/), syncs your transactions into a structured database, and hands that data to the AI services you already use through a built-in <Tooltip tip="Model Context Protocol — the open standard for giving AI agents access to tools and data. See modelcontextprotocol.io">[MCP](https://modelcontextprotocol.io)</Tooltip> server and a REST API.

Breadbox is not meant to be your financial dashboard, it is the infrastructure that can power your financial dashboard for the post-AI world.

## Why Breadbox?

The best financial advisor is already the AI you use daily — an agent that knows you and your family has the context to help with categorization, budgeting, and anomaly detection. It just needs structured access to your data without your bank credentials.

Breadbox bridges that gap: secure bank sync, a database you control, and a clean queryable interface for any tool or agent.

## Features

- **Bank sync** via Plaid, Teller, and CSV import (with more coming)
- **MCP server** for AI-agent integration — Streamable HTTP at `/mcp` and stdio
- **REST API** with cursor pagination, field selection, and filtering
- **Admin dashboard** for connection management, sync monitoring, and transaction review
- **Transaction rules engine** with recursive AND/OR/NOT conditions for self-improving data enrichment
- **Category system** with a two-level hierarchy
- **Tag-based workflows** — tags allow flexible coordination between humans and agents (e.g., the seeded `needs-review` tag lets an agent flag transactions for you to look at)
- **Account linking** to deduplicate transactions across connections (e.g., dependent cards)
- **Agent reports** for AI agents to submit summaries and flag transactions
- **API key auth** with scoped access (full access or read-only)
- **AES-256-GCM encryption** for all provider credentials stored at rest
- **Single binary** — one Go binary serves everything: API, MCP server, admin dashboard, webhooks, and the sync scheduler

## What you need to get started

Before installing Breadbox, make sure you have the following:

| Requirement | Details |
| --- | --- |
| **Docker** (recommended) | Docker Engine \+ Docker Compose, or any Linux/macOS machine |
| **PostgreSQL** | Version 16 or later. Docker Compose handles this automatically. |
| **Bank API credentials** | A [Plaid](https://plaid.com/) or a [Teller](https://teller.io/) API keys — or both |

You can also configure Plaid and Teller credentials after installation through the setup wizard — you don't need them before you start.
