Breadbox is an open-source, self-hosted financial aggregator for households. It syncs your bank transactions into a PostgreSQL database you own and exposes them through a REST API and a built-in MCP server for AI agents.

## Run Breadbox in three commands

Spin up Breadbox and PostgreSQL with Docker Compose:

```bash
curl -O https://raw.githubusercontent.com/canalesb93/breadbox/main/docker-compose.yml
cp .env.example .env
docker compose up -d breadbox db
```

Once the containers are healthy, open your Breadbox URL and the first-run wizard will walk you through creating your admin account, configuring a bank provider, and setting your sync interval.

<Warning>
  **Plan for remote access from the start.** Breadbox's full value — MCP access from Claude, ChatGPT, Manus, Openclaw; real-time webhooks from Plaid/Teller; access from any of your devices — depends on a publicly reachable HTTPS URL. Before you finish the wizard, read [Remote Hosting](/installation/remote-hosting) and pick a host (exe.dev, Cloudflare Tunnel, Caddy on a VPS, Tailscale Funnel).

  Running Breadbox only on `http://localhost:8080` makes sense in one narrow case: when a local autonomous agent (e.g., an Openclaw runner on the same machine) is the *only* client that will ever need to reach it. Otherwise, go remote.
</Warning>
