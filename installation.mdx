Breadbox ships as a single Go binary that bundles the REST API, MCP server, admin dashboard, webhooks, and sync scheduler. The recommended setup is a single command that handles Docker, secrets, optional HTTPS, and starts the stack for you.

<Card title="Planning to use MCP or bank webhooks?" icon="globe" href="/installation/remote-hosting">
  Those need a public HTTPS URL, not `localhost`. See the remote-hosting guide for exe.dev, Cloudflare Tunnel, Caddy-on-a-VPS, and Tailscale Funnel walk-throughs.
</Card>

## Quick install

Run this on the machine where you want Breadbox to live (a VPS, a spare laptop, a NAS running Docker — anywhere `docker` can run):

```bash
curl -fsSL https://breadbox.sh/install.sh | bash
```

The installer detects your OS (macOS or Linux) and architecture (`amd64` / `arm64`), checks for Docker (and offers to install it on Linux via `get.docker.com` when missing), prompts for an optional public domain, generates an `ENCRYPTION_KEY` and database password, writes a version-pinned `docker-compose.prod.yml`, starts the stack, and — on success — offers to register a launchd agent (macOS) or systemd unit (Linux) so Breadbox restarts on boot. The final step runs `breadbox doctor` inside the container to verify the install.

<Note>
  `breadbox.sh/install.sh` is a thin bootstrap shim that fetches the real installer from the `canalesb93/breadbox` GitHub repo. If you'd rather skip the shim (or if `breadbox.sh` isn't reachable), the direct URL works the same way:

  ```bash
  curl -fsSL https://raw.githubusercontent.com/canalesb93/breadbox/main/deploy/install.sh | bash
  ```
</Note>

### Non-interactive install

Skip the prompts and configure a public domain in one shot:

```bash
curl -fsSL https://breadbox.sh/install.sh | bash -s -- --yes --domain=breadbox.example.com
```

`--domain=<host>` enables the Caddy reverse proxy (HTTPS on :443 with automatic Let's Encrypt certs). Leave it off for a localhost-only install — the Caddy service is gated behind a compose profile and doesn't start when no domain is set, so ports 80/443 are not bound.

### Install directory

| Invocation | Install location |
| --- | --- |
| `sudo bash ...` (root) | `/opt/breadbox` |
| regular user | `$HOME/.breadbox` |
| override | `INSTALL_DIR=/custom/path bash install.sh` |

### Updates

`./update.sh` inside the install directory pulls the pinned version on every run. Change the pin explicitly:

```bash
./update.sh --bump=v0.4.0   # pin to a specific tag
./update.sh --bump=latest   # track latest (not recommended for production)
```

<Warning>
  **Remote hosting is the intended default.** Most of Breadbox's value — MCP access from Claude, ChatGPT, Manus, Openclaw, and other cloud AI clients, plus real-time webhooks from Plaid/Teller — requires a public HTTPS URL. Pass `--domain` to the installer (or come back and set `DOMAIN=` in `.env` later), then pick a host approach from the [remote-hosting guide](/installation/remote-hosting).

  The exception: if the only agent touching Breadbox is a local autonomous runner (e.g., Openclaw on the same machine), you can run on `http://localhost:8080` indefinitely.
</Warning>

## Alternative install methods

The one-liner above is the supported default. The methods below are for specific cases: integrating Breadbox into an existing Compose stack, running without Docker, or hacking on the source.

<Tabs>
  <Tab title="Docker Compose (manual)">
    Use this if you already manage a Compose stack and want to drop Breadbox into it yourself.

    <Steps>
      <Step title="Download the Compose file">
        ```bash
        mkdir breadbox && cd breadbox
        curl -O https://raw.githubusercontent.com/canalesb93/breadbox/main/deploy/docker-compose.prod.yml
        mv docker-compose.prod.yml docker-compose.yml
        ```
      </Step>
      <Step title="Create your .env file">
        ```bash
        cat > .env <<EOF
        DATABASE_URL=postgres://breadbox:breadbox@db:5432/breadbox?sslmode=disable
        ENCRYPTION_KEY=$(openssl rand -hex 32)
        SERVER_PORT=8080
        ENVIRONMENT=docker
        POSTGRES_USER=breadbox
        POSTGRES_PASSWORD=$(openssl rand -base64 32 | tr -d '=/+')
        POSTGRES_DB=breadbox
        EOF
        ```

        <Warning>
          Save your `.env` somewhere safe. The `ENCRYPTION_KEY` encrypts all stored bank credentials — lose it and you can't recover those credentials after the fact.
        </Warning>
      </Step>
      <Step title="Start Breadbox">
        Localhost-only (Caddy stays off):

        ```bash
        docker compose up -d breadbox db
        ```

        Or with the Caddy HTTPS reverse proxy (set `DOMAIN=` in `.env` first):

        ```bash
        docker compose --profile caddy up -d
        ```

        Caddy is gated behind the `caddy` compose profile — it only binds ports 80/443 when you opt in. Open `http://your-host:8080` (or your HTTPS domain) to hit the setup wizard.
      </Step>
    </Steps>

    Pin a specific version in `docker-compose.yml`:

    ```yaml
    image: ghcr.io/canalesb93/breadbox:v0.1.0
    ```
  </Tab>

  <Tab title="Binary">
    Pre-built binaries for Linux and macOS (amd64/arm64) are on the [GitHub Releases](https://github.com/canalesb93/breadbox/releases) page. You'll need a running PostgreSQL 16+ instance.

    ```bash
    # Linux amd64 example — swap for your platform
    curl -fsSL https://github.com/canalesb93/breadbox/releases/latest/download/breadbox-linux-amd64 -o breadbox
    chmod +x breadbox

    export DATABASE_URL="postgres://user:pass@localhost:5432/breadbox?sslmode=disable"
    export ENCRYPTION_KEY="$(openssl rand -hex 32)"

    ./breadbox serve
    # Visit http://your-host:8080
    ```
  </Tab>

  <Tab title="Go install">
    Build from source with Go 1.24 or later. You'll need a running PostgreSQL 16+ instance.

    ```bash
    git clone https://github.com/canalesb93/breadbox.git && cd breadbox
    go install ./cmd/breadbox

    export DATABASE_URL="postgres://user:pass@localhost:5432/breadbox?sslmode=disable"
    export ENCRYPTION_KEY="$(openssl rand -hex 32)"

    breadbox serve
    # Visit http://your-host:8080
    ```

    <Note>
      The module path is `breadbox` (not `github.com/...`), so `go install github.com/canalesb93/breadbox/cmd/breadbox@latest` is not supported. Clone the repo and `go install` locally.
    </Note>
  </Tab>
</Tabs>

## Environment variables

Most Breadbox configuration is done through environment variables. Set these in your `.env` file or export them directly before running the binary. The table below covers the essentials; the [environment variables reference](/configuration/environment) documents every variable, precedence rules, and how env vars interact with dashboard settings.

| Variable | Required | Description |
| --- | --- | --- |
| `DATABASE_URL` | Yes | PostgreSQL connection string, e.g. `postgres://user:pass@host:5432/breadbox?sslmode=disable` |
| `ENCRYPTION_KEY` | Yes | 32-byte key as a 64-character hex string. Generate with `openssl rand -hex 32`. Required when any bank provider is configured. |
| `SERVER_PORT` | No | HTTP listen port. Defaults to `8080`. |
| `DOMAIN` | No | Public hostname (e.g. `breadbox.example.com`). Enables the `caddy` compose profile for automatic HTTPS; leave unset for localhost-only installs. |
| `PLAID_CLIENT_ID` | No | Plaid API client ID. Can also be set through the setup wizard. |
| `PLAID_SECRET` | No | Plaid API secret. Can also be set through the setup wizard. |
| `TELLER_APP_ID` | No | Teller application ID. Can also be set through the setup wizard. |

<Tip>
  You can leave `PLAID_CLIENT_ID`, `PLAID_SECRET`, and `TELLER_APP_ID` out of your `.env` file and configure them later through the admin setup wizard.
</Tip>

## Verify the install

Run `breadbox doctor` at any time to check config and connectivity without booting the server:

```bash
# Docker install: exec into the container
docker compose exec breadbox breadbox doctor

# Binary / go install: run against your env directly
breadbox doctor
```

The command walks a fixed list of pre-flight checks — config-load, database connection, migrations, `ENCRYPTION_KEY`, Plaid/Teller configuration, bank-connection credentials decrypting, an admin account exists, scheduler cron expressions parse, and (when `PUBLIC_URL`/`DOMAIN` is set) your public URL resolves and `/health/ready` responds. Exits non-zero if anything fails so install scripts and CI can gate on it.

Handy flags:

- `--json` — structured output for scripting.
- `--skip-external` — skip DNS and outbound HTTP (air-gapped installs).

## After installing

Once Breadbox is running, open `http://your-host:8080` (or your HTTPS domain if you set one) to complete the first-run setup wizard. The wizard walks you through creating your admin account and configuring bank provider credentials.
