This guide takes you from a fresh install to your first queried transaction. By the end, you'll have Breadbox running, a bank account connected, and your transactions accessible through <Tooltip tip="Model Context Protocol — modelcontextprotocol.io">[MCP](https://modelcontextprotocol.io)</Tooltip> (for AI clients like Claude, ChatGPT, or Openclaw) and the REST API.

## Step 1: Start Breadbox with Docker Compose

If you haven't installed Breadbox yet, follow the [installation guide](/installation) first. Once your `.env` file is ready, start Breadbox and PostgreSQL:

```bash
docker compose up -d breadbox db
```

Wait a few seconds for the database to initialize and for Breadbox to run its migrations. You can check that everything is healthy:

```bash
docker compose ps
```

Both the `breadbox` and `db` services should show a `running` or `healthy` status.

## Step 2: Open the setup wizard

Open your Breadbox URL. If this is your first launch, Breadbox redirects you to the setup wizard automatically. The wizard runs once and collects everything Breadbox needs before you can connect a bank.

<Warning>
  **Configure remote hosting before (or during) the wizard.** Breadbox is designed to be reached from the cloud AI clients in your life (Claude, ChatGPT, Manus, Openclaw) and to receive real-time webhooks from Plaid/Teller — both of which require a public HTTPS URL. If you're just running a one-off evaluation on `http://localhost:8080`, that's fine; but plan on standing up remote hosting before you call the install "done." See [Remote Hosting](/installation/remote-hosting) for exe.dev, Cloudflare Tunnel, Caddy, and Tailscale Funnel walk-throughs.

  The one legitimate localhost-only case is running Breadbox alongside a *local* autonomous agent (e.g., an Openclaw runner on the same box) that never needs external reach.
</Warning>

## Step 3: Complete the setup wizard

The setup wizard has five steps. Work through them in order:

<Steps>
  <Step title="Create your admin account">
    Enter a username and password for your admin account. This is the account you'll use to log in to the dashboard and manage connections.
  </Step>
  <Step title="Configure your bank provider">
    Enter your Plaid or Teller API credentials.

    - **Plaid:** Paste your **Client ID** and **Secret** from the [Plaid dashboard](https://dashboard.plaid.com/). Set the environment to `sandbox` for testing.
    - **Teller:** Enter your **App ID** from the [Teller console](https://teller.io/).

    <Note>
      If you're testing, use Plaid's sandbox environment. It's free and includes pre-populated test institutions with realistic transaction history — no production approval needed.
    </Note>
  </Step>
  <Step title="Set the sync interval">
    Choose how often Breadbox polls your banks for new transactions. The default is every 12 hours. You can change this at any time from the admin dashboard.
  </Step>
  <Step title="Set a webhook URL">
    Enter the public URL of your Breadbox instance (e.g., `https://your-breadbox.example.com`). Plaid and Teller send real-time webhooks to `https://<your-host>/webhooks/<provider>` so transactions arrive in seconds instead of on the polling interval.

    You can leave this blank temporarily if you're still setting up remote hosting — polling-only sync works without it — but come back and fill it in once your public URL is live. See [Remote Hosting](/installation/remote-hosting).
  </Step>
  <Step title="Finish">
    Review your settings and click **Complete setup**. Breadbox saves your configuration and redirects you to the admin dashboard.
  </Step>
</Steps>

## Step 4: Connect a bank

From the admin dashboard, navigate to **Connections** and click **Add connection**.

Breadbox opens the Plaid Link (or Teller) widget in your browser. Select your bank, sign in with your bank credentials, and grant access to your accounts. Your credentials go directly to Plaid or Teller — Breadbox never sees them.

After you complete the widget flow, Breadbox stores the encrypted access token and queues an initial sync for that connection.

## Step 5: Wait for the initial sync

The initial sync fetches your full transaction history from your bank. For most connections this takes under a minute, but it can take longer for accounts with several years of history.

Watch the sync status from the dashboard under **Connections**. The status changes from `in_progress` to `success` when the sync completes.

<Tip>
  You can also trigger a manual sync at any time from the **Connections** page by clicking **Sync now** next to a connection.
</Tip>

## Step 6: Query your first transaction

Once the sync completes, use the REST API to fetch your transactions. You'll need an API key — generate one from **Settings → API keys** in the admin dashboard.

```bash
curl -H "X-API-Key: bb_your_api_key" \
  "https://your-breadbox.example.com/api/v1/transactions?limit=5"
```

Replace `your-breadbox.example.com` with your actual host. If you're evaluating on the same machine, `http://localhost:8080` works — but plan to expose Breadbox publicly before integrating with any cloud AI client.

A successful response looks like this:

```json
{
  "data": [
    {
      "id": "txn_abc12345",
      "account_id": "acc_xyz98765",
      "amount": "42.50",
      "iso_currency_code": "USD",
      "name": "WHOLE FOODS MARKET",
      "merchant_name": "Whole Foods",
      "date": "2026-04-18",
      "pending": false,
      "category": {
        "primary": "Food and Drink",
        "detailed": "Groceries"
      }
    }
  ],
  "next_cursor": "eyJpZCI6InR4bl9hYmMxMjM0NSJ9"
}
```

<Note>
  Replace `bb_your_api_key` with the key you generated from the dashboard. API keys always start with the `bb_` prefix.
</Note>

## Connect an agent

The REST API is only half the story — Breadbox ships with a built-in MCP server so AI assistants can query and enrich your data directly. Point Claude, Cursor, or any MCP-compatible client at `/mcp` (Streamable HTTP) or run `breadbox mcp-stdio` for stdio transports. See [MCP setup](/mcp/setup) for client-specific instructions, and browse the [Guides](/guides/overview) for end-to-end agent workflows like weekly categorization reviews and anomaly detection.
