Breadbox ships as a single Go binary that bundles the REST API, MCP server, admin dashboard, webhooks, and sync scheduler. Pick the install method you're most comfortable with below — they all land at the same first-run setup wizard.

<Card title="Planning to use MCP or bank webhooks?" icon="globe" href="/installation/remote-hosting">
  Those need a public HTTPS URL, not `localhost`. See the remote-hosting guide for exe.dev, Cloudflare Tunnel, Caddy-on-a-VPS, and Tailscale Funnel walk-throughs.
</Card>

<Tabs>
  <Tab title="Docker Compose">
    Docker Compose is the fastest way to run Breadbox. It starts both Breadbox and PostgreSQL for you — no separate database setup required.

    <Steps>
      <Step title="Download the Compose file">
        Create a directory for Breadbox and download the production Compose file:

        ```bash
        mkdir breadbox && cd breadbox
        curl -O https://raw.githubusercontent.com/canalesb93/breadbox/main/deploy/docker-compose.prod.yml
        mv docker-compose.prod.yml docker-compose.yml
        ```
      </Step>
      <Step title="Create your .env file">
        Generate a `.env` file with the required configuration. The commands below generate a random encryption key and database password for you:

        ```bash
        cat > .env <<EOF
        DATABASE_URL=postgres://breadbox:breadbox@db:5432/breadbox?sslmode=disable
        ENCRYPTION_KEY=$(openssl rand -hex 32)
        SERVER_PORT=8080
        ENVIRONMENT=docker
        POSTGRES_USER=breadbox
        POSTGRES_PASSWORD=$(openssl rand -base64 32 | tr -d '=/+')
        POSTGRES_DB=breadbox
        EOF
        ```

        <Warning>
          Save your `.env` file somewhere safe. The `ENCRYPTION_KEY` encrypts all stored bank credentials. If you lose it after connecting a bank, you cannot recover those credentials.
        </Warning>
      </Step>
      <Step title="Start Breadbox">
        Start Breadbox and PostgreSQL:

        ```bash
        docker compose up -d breadbox db
        ```

        This starts both services and runs any pending database migrations automatically. Open `http://your-host:8080` to access the setup wizard.
      </Step>
    </Steps>

    <Note>
      This command starts Breadbox and PostgreSQL without the Caddy reverse proxy, exposing port 8080 directly. For a production server with automatic HTTPS, see [Production deployment](#production-deployment) below or the [remote-hosting guide](/installation/remote-hosting).
    </Note>

    To pin a specific version instead of `latest`, edit `docker-compose.yml` and change the image tag:

    ```yaml
    image: ghcr.io/canalesb93/breadbox:v0.1.0
    ```
  </Tab>

  <Tab title="Binary download">
    Pre-built binaries for Linux and macOS (amd64/arm64) are available on the [GitHub Releases](https://github.com/canalesb93/breadbox/releases) page. You'll need a running PostgreSQL 16+ instance before starting.

    ```bash
    # Download the binary for your platform (example: Linux amd64)
    curl -fsSL https://github.com/canalesb93/breadbox/releases/latest/download/breadbox-linux-amd64 -o breadbox
    chmod +x breadbox

    # Set required environment variables
    export DATABASE_URL="postgres://user:pass@localhost:5432/breadbox?sslmode=disable"
    export ENCRYPTION_KEY="$(openssl rand -hex 32)"

    ./breadbox serve
    # Visit http://your-host:8080
    ```
  </Tab>

  <Tab title="Go install">
    Build from source using Go 1.24 or later. You'll need a running PostgreSQL 16+ instance.

    ```bash
    git clone https://github.com/canalesb93/breadbox.git && cd breadbox
    go install ./cmd/breadbox

    # Set required environment variables
    export DATABASE_URL="postgres://user:pass@localhost:5432/breadbox?sslmode=disable"
    export ENCRYPTION_KEY="$(openssl rand -hex 32)"

    breadbox serve
    # Visit http://your-host:8080
    ```

    <Note>
      The module path is `breadbox`, so `go install github.com/...@latest` is not supported. Clone the repo and install locally as shown above.
    </Note>
  </Tab>

  <Tab title="One-liner (remote server)">
    For a production server with automatic HTTPS via Caddy, use the one-liner install script. It installs Docker if needed, downloads the deployment files, generates secrets, and starts Breadbox with Caddy for automatic TLS:

    ```bash
    curl -fsSL https://raw.githubusercontent.com/canalesb93/breadbox/main/deploy/install.sh | sudo bash
    ```

    After the script completes, set your domain in `breadbox/.env` (`DOMAIN=breadbox.example.com`) and start all services including the Caddy reverse proxy:

    ```bash
    docker compose up -d
    ```

    See [Production deployment](#production-deployment) for the full walk-through, or the [remote-hosting guide](/installation/remote-hosting) for alternatives like Cloudflare Tunnel.
  </Tab>
</Tabs>

<Warning>
  **Remote hosting is the intended default.** Most of Breadbox's value — MCP access from Claude, ChatGPT, Manus, Openclaw, and other cloud AI clients, plus real-time webhooks from Plaid/Teller — requires a public HTTPS URL. Pick a host from the [remote-hosting guide](/installation/remote-hosting) (exe.dev, Cloudflare Tunnel, Caddy on a VPS, Tailscale Funnel) and do that setup alongside this install.

  The exception: if the only agent touching Breadbox is a local autonomous runner (e.g., Openclaw on the same machine), you can run on `http://localhost:8080` indefinitely.
</Warning>

## Production deployment

For a production server with automatic HTTPS via Caddy, the one-liner installer wires up Docker, Breadbox, PostgreSQL, and Caddy in one shot:

```bash
curl -fsSL https://raw.githubusercontent.com/canalesb93/breadbox/main/deploy/install.sh | sudo bash
```

After the script completes, configure your domain in the Caddyfile (via the `DOMAIN` variable in `.env`) and start all services including the Caddy reverse proxy:

```bash
docker compose up -d
```

If you don't have a VPS (or don't want to open port 443 on a home network), the [remote-hosting guide](/installation/remote-hosting) covers tunnel-based alternatives like Cloudflare Tunnel, exe.dev, and Tailscale Funnel.

## Environment variables

All Breadbox configuration is done through environment variables. Set these in your `.env` file or export them directly before running the binary.

| Variable | Required | Description |
| --- | --- | --- |
| `DATABASE_URL` | Yes | Full PostgreSQL connection string, e.g. `postgres://user:pass@host:5432/breadbox?sslmode=disable` |
| `ENCRYPTION_KEY` | Yes | 32-byte key as a 64-character hex string. Generate with `openssl rand -hex 32`. Required when any bank provider is configured. |
| `SERVER_PORT` | No | HTTP listen port. Defaults to `8080`. |
| `PLAID_CLIENT_ID` | No | Plaid API client ID. Can also be set through the setup wizard. |
| `PLAID_SECRET` | No | Plaid API secret. Can also be set through the setup wizard. |
| `TELLER_APP_ID` | No | Teller application ID. Can also be set through the setup wizard. |

<Tip>
  You can leave `PLAID_CLIENT_ID`, `PLAID_SECRET`, and `TELLER_APP_ID` out of your `.env` file and configure them later through the admin setup wizard.
</Tip>

## After installing

Once Breadbox is running, open `http://your-host:8080` (or your HTTPS domain if you've set one up) to complete the setup wizard. The wizard walks you through creating your admin account and configuring your bank provider credentials before you connect any banks.
