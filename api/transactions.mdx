Transactions are the core data in Breadbox. The transactions API lets you query with rich filters, count and summarize spending, retrieve merchant statistics, and set or reset category overrides — individually or in bulk. All amounts follow the Plaid sign convention: positive is money out (debits), negative is money in (credits).

---

## List transactions

```text
GET /api/v1/transactions
```

Returns a paginated list of transactions sorted by date descending by default. Soft-deleted transactions are always excluded. All filters are combined with `AND` logic.

**Auth:** Read

### Query parameters

<ParamField query="account_id" type="string (UUID)">
  Filter to transactions from a single account.
</ParamField>

<ParamField query="user_id" type="string (UUID)">
  Filter to transactions from all accounts owned by a family member. Uses attribution-aware logic: `COALESCE(attributed_user_id, account.user_id)`.
</ParamField>

<ParamField query="category_slug" type="string">
  Filter by category slug (e.g., `food-and-drink-groceries`).
</ParamField>

<ParamField query="start_date" type="string (YYYY-MM-DD)">
  Inclusive lower bound on the transaction date.
</ParamField>

<ParamField query="end_date" type="string (YYYY-MM-DD)">
  Exclusive upper bound on the transaction date. A transaction on `end_date` is excluded — use this to express calendar months without overlap (e.g., `start_date=2025-01-01&end_date=2025-02-01` returns all of January).
</ParamField>

<ParamField query="min_amount" type="number">
  Inclusive lower bound on amount. Uses Plaid convention (positive = debit). To find all debits over $100: `min_amount=100`.
</ParamField>

<ParamField query="max_amount" type="number">
  Inclusive upper bound on amount. To find all credits: `max_amount=-0.01`.
</ParamField>

<ParamField query="search" type="string">
  Case-insensitive search on `name` and `merchant_name`. Comma-separate values for OR logic. Minimum 2 characters.
</ParamField>

<ParamField query="exclude_search" type="string">
  Exclude transactions where `name` or `merchant_name` matches this term. Minimum 2 characters.
</ParamField>

<ParamField query="search_mode" type="string" default="contains">
  How the `search` term is matched. One of: `contains` (default substring), `words` (word-boundary match), `fuzzy` (trigram similarity).
</ParamField>

<ParamField query="pending" type="boolean">
  `true` returns only pending transactions; `false` returns only posted. Omit to return both.
</ParamField>

<ParamField query="tags" type="string">
  Comma-separated list of tag slugs. Returns only transactions that carry **every** listed tag (AND semantics). Example: `tags=needs-review,subscription`. See the [Tags API](/api/tags) for the full tag-management surface.
</ParamField>

<ParamField query="any_tag" type="string">
  Comma-separated list of tag slugs. Returns transactions that carry **at least one** of the listed tags (OR semantics). Example: `any_tag=flagged,disputed`.
</ParamField>

<ParamField query="sort_by" type="string" default="date">
  Sort field. One of: `date`, `amount`, `name`. Cursor pagination only works with `date` sort.
</ParamField>

<ParamField query="sort_order" type="string" default="desc">
  Sort direction. One of: `desc`, `asc`.
</ParamField>

<ParamField query="fields" type="string">
  Field selection preset. One of: `minimal`, `core`, `category`, `timestamps`. Returns a subset of fields to reduce response size.
</ParamField>

<ParamField query="cursor" type="string">
  Opaque pagination cursor from a previous response. Omit to fetch the first page. Only works with `date` sort.
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Results per page. Minimum `1`, maximum `500`.
</ParamField>

## Count transactions

```text
GET /api/v1/transactions/count
```

Returns the total number of transactions matching the given filters, without fetching the transaction records themselves. Accepts the same filter parameters as the list endpoint (excluding `cursor`, `limit`, `sort_by`, `sort_order`, and `fields`).

**Auth:** Read

### Response

```json
{
  "count": 1523
}
```

```bash
curl -H "X-API-Key: bb_your_key" \
  "http://localhost:8080/api/v1/transactions/count?start_date=2025-01-01&end_date=2026-01-01"
```

## Summarize transactions

```text
GET /api/v1/transactions/summary
```

Returns aggregated spending totals grouped by category, month, week, or day. Useful for building spending breakdowns and charts without pulling every transaction record.

**Auth:** Read

Accepts the same filter parameters as the list endpoint. The `group_by` parameter (required — one of `category`, `month`, `week`, `day`, `category_month`) controls the aggregation dimension.

### Response

```json
{
  "summary": [
    {
      "category": "food-and-drink",
      "total_amount": 843.21,
      "transaction_count": 34,
      "iso_currency_code": "USD"
    },
    {
      "category": "transportation",
      "total_amount": 312.50,
      "transaction_count": 18,
      "iso_currency_code": "USD"
    }
  ],
  "totals": {
    "total_amount": 1155.71,
    "transaction_count": 52
  },
  "filters": {
    "start_date": "",
    "end_date": "",
    "group_by": "category"
  }
}
```

Rows aggregated by `month`, `week`, or `day` include a `period` string (e.g., `"2026-02"`) instead of `category`. Rows aggregated by `category_month` include both fields.

## Set category override

```text
PATCH /api/v1/transactions/{id}/category
```

Sets a manual category override on a single transaction. Once overridden, Breadbox rules and future syncs will not change the category — the override is permanent until explicitly deleted. Overridden transactions have `category_override: true` in their response.

**Auth:** Write

### Path parameters

<ParamField path="id" type="string" required>
  The transaction's Breadbox UUID or short ID.
</ParamField>

### Request body

```json
{
  "category_id": "food-and-drink-groceries"
}
```

<ParamField body="category_id" type="string" required>
  Identifier of the category to assign. Accepts a category UUID, 8-character short ID, or slug (e.g., `food-and-drink-groceries`). Must match an existing category.
</ParamField>

### Response

Returns `204 No Content` on success.

```bash
curl -X PATCH \
  -H "X-API-Key: bb_your_key" \
  -H "Content-Type: application/json" \
  -d '{"category_id": "food-and-drink-groceries"}' \
  "http://localhost:8080/api/v1/transactions/t1a2b3c4-d5e6-f789-0abc-def123456789/category"
```

## Reset category override

```text
DELETE /api/v1/transactions/{id}/category
```

Removes a manual category override, restoring the transaction to its provider-assigned category. After deletion, rules and future syncs may re-categorize the transaction normally.

**Auth:** Write

### Path parameters

<ParamField path="id" type="string" required>
  The transaction's Breadbox UUID or short ID.
</ParamField>

### Response

Returns `204 No Content` on success.

```bash
curl -X DELETE \
  -H "X-API-Key: bb_your_key" \
  "http://localhost:8080/api/v1/transactions/t1a2b3c4-d5e6-f789-0abc-def123456789/category"
```

## Batch categorize

```text
POST /api/v1/transactions/batch-categorize
```

Sets category overrides on up to 500 transactions in a single request. Each item pairs a transaction with its target category, so you can assign different categories in one call.

**Auth:** Write

### Request body

```json
{
  "items": [
    { "transaction_id": "t1a2b3c4-d5e6-f789-0abc-def123456789", "category_slug": "food-and-drink-groceries" },
    { "transaction_id": "t2b3c4d5-e6f7-890a-bcde-f12345678901", "category_slug": "transportation-gas" }
  ]
}
```

<ParamField body="items" type="array of objects" required>
  Up to 500 `{transaction_id, category_slug}` pairs. `transaction_id` accepts either a UUID or an 8-character short ID.
</ParamField>

### Response

```json
{
  "succeeded": 2,
  "failed": [
    { "transaction_id": "t3...unknown", "error": "transaction not found" }
  ]
}
```

The `failed` array is omitted when every item succeeded.

```bash
curl -X POST \
  -H "X-API-Key: bb_your_key" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      { "transaction_id": "t1a2b3c4-d5e6-f789-0abc-def123456789", "category_slug": "food-and-drink-groceries" }
    ]
  }' \
  "http://localhost:8080/api/v1/transactions/batch-categorize"
```

## Bulk recategorize

```text
POST /api/v1/transactions/bulk-recategorize
```

Recategorizes all transactions matching a set of filters in a single server-side operation. Unlike batch categorize, you do not need to know the individual transaction IDs. This endpoint is useful for correcting a category across a large set of matching transactions at once.

**Auth:** Write

### Request body

```json
{
  "target_category_slug": "entertainment-streaming-services",
  "search": "NETFLIX",
  "start_date": "2025-01-01"
}
```

<ParamField body="target_category_slug" type="string" required>
  The category slug to assign to every transaction that matches the filters.
</ParamField>

<ParamField body="start_date" type="string (YYYY-MM-DD)">
  Inclusive lower bound on the transaction date.
</ParamField>

<ParamField body="end_date" type="string (YYYY-MM-DD)">
  Inclusive upper bound on the transaction date.
</ParamField>

<ParamField body="account_id" type="string">
  Limit to a specific account (UUID or short ID).
</ParamField>

<ParamField body="user_id" type="string">
  Limit to transactions attributed to a specific family member.
</ParamField>

<ParamField body="category_slug" type="string">
  Filter by the transaction's **current** category slug (distinct from `target_category_slug`, which is the replacement).
</ParamField>

<ParamField body="min_amount" type="number">
  Inclusive lower bound on amount (Plaid sign convention).
</ParamField>

<ParamField body="max_amount" type="number">
  Inclusive upper bound on amount.
</ParamField>

<ParamField body="pending" type="boolean">
  Filter to pending (`true`) or posted (`false`) transactions.
</ParamField>

<ParamField body="search" type="string">
  Case-insensitive search on `name` and `merchant_name`.
</ParamField>

<ParamField body="name_contains" type="string">
  Case-insensitive substring match against `name` only (ignores `merchant_name`).
</ParamField>

### Response

```json
{
  "matched_count": 14,
  "updated_count": 14
}
```

`matched_count` is the number of transactions that matched the filters; `updated_count` is the number actually modified (excludes transactions that already had `category_override` set, which are preserved).

## Transaction response fields

<ResponseField name="id" type="string (UUID)" required>
  Breadbox-assigned UUID for this transaction.
</ResponseField>

<ResponseField name="short_id" type="string" required>
  8-character base62 alias. Accepted interchangeably with `id` in path parameters.
</ResponseField>

<ResponseField name="account_id" type="string (UUID)">
  UUID of the account this transaction belongs to. May be `null` if the parent account was removed.
</ResponseField>

<ResponseField name="external_transaction_id" type="string" required>
  The transaction ID assigned by the provider (e.g., Plaid's `transaction_id`). Unique within a provider item.
</ResponseField>

<ResponseField name="pending_transaction_id" type="string">
  If this posted transaction replaced a pending one, this field holds the `external_transaction_id` of the original pending transaction. Use this to link posted and pending records. `null` otherwise.
</ResponseField>

<ResponseField name="amount" type="number" required>
  Transaction amount. Positive = money out (debit). Negative = money in (credit). Stored as `NUMERIC(12,2)`.
</ResponseField>

<ResponseField name="iso_currency_code" type="string">
  ISO 4217 currency code (e.g., `"USD"`). `null` when an unofficial currency code is used.
</ResponseField>

<ResponseField name="date" type="string (YYYY-MM-DD)" required>
  Posted date for settled transactions; pending date for pending transactions.
</ResponseField>

<ResponseField name="datetime" type="string (ISO 8601)">
  Full timestamp of the transaction, if provided by the institution. `null` for most transactions.
</ResponseField>

<ResponseField name="authorized_date" type="string (YYYY-MM-DD)">
  Date the transaction was authorized. May differ from `date`. `null` when not provided.
</ResponseField>

<ResponseField name="name" type="string" required>
  Raw transaction name from the institution (e.g., `"WHOLE FOODS MARKET #10452"`).
</ResponseField>

<ResponseField name="merchant_name" type="string">
  Cleaned merchant name as classified by the provider (e.g., `"Whole Foods Market"`). `null` if the provider could not identify the merchant.
</ResponseField>

<ResponseField name="category_primary" type="string">
  High-level category in `UPPER_SNAKE_CASE` (e.g., `"FOOD_AND_DRINK"`). May reflect a manual override.
</ResponseField>

<ResponseField name="category_detailed" type="string">
  Granular subcategory (e.g., `"FOOD_AND_DRINK_GROCERIES"`). May reflect a manual override.
</ResponseField>

<ResponseField name="category_override" type="boolean" required>
  `true` if the category was set by a manual override and is protected from rule and sync changes.
</ResponseField>

<ResponseField name="payment_channel" type="string">
  How the transaction was made. One of: `"in store"`, `"online"`, `"other"`. `null` when not reported.
</ResponseField>

<ResponseField name="pending" type="boolean" required>
  `true` if the transaction has not yet posted. Pending transactions may be updated or replaced.
</ResponseField>

## Error codes

| Condition | Status | Code |
| --- | --- | --- |
| Missing or invalid API key | `401` | `MISSING_API_KEY` / `INVALID_API_KEY` |
| `limit` out of range | `400` | `INVALID_PARAMETER` |
| Date not in `YYYY-MM-DD` format | `400` | `INVALID_PARAMETER` |
| `start_date` is after `end_date` | `400` | `INVALID_PARAMETER` |
| `min_amount` greater than `max_amount` | `400` | `INVALID_PARAMETER` |
| Invalid UUID for `account_id` or `user_id` | `400` | `INVALID_PARAMETER` |
| `search` shorter than 2 characters | `400` | `INVALID_PARAMETER` |
| `search_mode` not one of `contains` / `words` / `fuzzy` | `400` | `INVALID_PARAMETER` |
| `sort_order` not `asc` or `desc` | `400` | `INVALID_PARAMETER` |
| `group_by` missing or invalid (`/summary`) | `400` | `INVALID_PARAMETER` |
| Unknown field name in `fields=` | `400` | `INVALID_FIELDS` |
| Malformed or expired `cursor` | `400` | `INVALID_CURSOR` |
| Transaction not found | `404` | `NOT_FOUND` |
| Category not found (set / bulk-recategorize) | `404` | `CATEGORY_NOT_FOUND` |
